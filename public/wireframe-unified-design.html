Wiki Codex — Unified Design System

Proposal для единой дизайн-системы: типографика, цвета, spacing, theme-поддержка. Нажми ☀/🌙 чтобы переключить тему и увидеть проблемы.

1. Code Blocks — Dark-Only → Theme-Adaptive

CRITICAL — 5 files

Проблема: CopyableCodeBlock, semantic-highlight, step-type-config, step-card — всё захардкожено под тёмную тему. В light режиме код нечитаем.

Решение: Code block использует semantic tokens (bg-muted / bg-card), а syntax highlighting переключается через CSS custom properties.

❌ Before — dark-only
Установить зависимости
// Открой в браузере POST /api/backup "https://example.com" --verbose DATABASE_URL=postgres://...

⚡ В light теме — белый текст на белом фоне

✅ After — theme-adaptive
Установить зависимости
// Открой в браузере POST /api/backup "https://example.com" --verbose DATABASE_URL=postgres://...

✓ Работает в обеих темах через CSS variables

3. Syntax Highlighting — CSS Custom Properties

HIGH — semantic-highlight.tsx

Проблема: Все 9 типов токенов используют zinc/lime/purple/amber/green/blue/cyan-300/400 shade — невидимые на белом фоне.

Решение: Определить CSS variables для каждого типа токена, с разными значениями для light/dark.

--syntax-number#ea580c#fb923c1. step

5. Accent Color Tokens

HIGH — 30+ instances

Проблема: text-green-600 dark:text-green-400 повторяется 12 раз. text-amber-500 — 8 раз. Нет единого токена.

Решение: Определить CSS variables для accent цветов, с light/dark значениями. Использовать утилитарный класс text-accent-green вместо пары.

TokenLightDarkUsage
--accent-green #16a34a #4ade80 Prompt $, active nav, terminal cursor
--accent-amber #d97706 #fbbf24 Stars, warnings, AI sparkles
--accent-violet #7c3aed #a78bfa Semantic search, AI badges
--accent-coral #FA3913 #ff5a38 NEURO brand, dividers
--accent-emerald #059669 #34d399 DB instruction badges, success

6. Typography Scale — Standardize

MEDIUM — 22 instances

Проблема: Микро-размеры text-[9px], text-[10px], text-[11px], text-[13px] разбросаны повсюду без системы.

Решение: Утвердить 3 уровня: mono-label (10px), mono-badge (11px), mono-body (13px). Использовать Tailwind arbitrary values с CSS variables.

TokenSizeUsageCurrent mess
--text-micro10pxSection headers, counts, indicatorstext-[9px], text-[10px]
--text-label11pxTerminal title bar, breadcrumbs, labelstext-[11px]
--text-code13pxCode block body texttext-[13px]

Font Convention

font-mono — Terminal chrome: labels, badges, buttons, metadata, footer, sidebar, prompts

font-sans — Readable text: headings, descriptions, body content, card text

font-sans на h1-h3 — обязательно. Headings читаются лучше в sans-serif.

7. Spacing — Standardize TerminalFrame Pattern

MEDIUM — 12 inconsistencies

Проблема: TerminalFrame margins варьируются: m-4 md:m-6, m-3 sm:m-4 md:m-6, m-3 sm:m-4. Внутренний padding: p-3, p-3 sm:p-4, p-4.

Решение: Единый паттерн для всех views:

TerminalFrame margin: className="m-3 sm:m-4 md:m-6"

Inner padding: className="p-3 sm:p-4"
